!TOPIC 1023 TextEditor
!NOINDEX
!NOWRAP
1022OpEditor.TextEditor = object(1815MemoFile)
  ...
  teSearchSt : string[MaxSearchLen];     {String to search for}
  teReplaceSt : string[MaxSearchLen];    {String to replace it with}
  teOptionSt : string[MaxSearchOptions]; {Search options}
  teLastSearch : teSearchType;           {Type of last search operation}
  teReplacements : Word;                 {Number of replacements made}
  teBlockIndent : Byte;                  {Indent level for blocks}
  teLPT : Byte;                          {line printer (1-3)}
  teDefExt : ExtStr;                     {Default extension}
  teBlkBegin : MarkerRec;                {Start of block}
  teBlkEnd : MarkerRec;                  {End of block}
  ...
end;
TextEditorPtr = ^TextEditor;
!WRAP

Derived from the MemoFile, the TextEditor provides a host of new editing
commands, and the result is a plug-and-play editor comparable in power to
the Turbo Pascal editor. Most of the new commands fall into one of four
categories: file commands (for reading and writing text files), block
commands (copy, move, delete, read, write, etc.), text marker commands (for
setting place markers and returning to them), and search-and-replace
commands (for locating text and making one-to-one string substitutions
either locally or globally).

The data fields shown above should not be modified directly. They are
documented here primarily to assist in the writing of status routines and
menu-driven interfaces (see DESKPOP for examples).

teSearchSt is the last search string. teReplaceSt is the last replacement
string. teOptionSt contains the search options last used. teLastSearch
indicates the type of the last search operation. teReplacements indicates
the number of replacements that were made in the last search-and-replace
operation. A user-written status routine may want to report this
information when it detects that the teSearchEnd flag is set. teBlockIndent
is the indentation level used for the ccBlkIndent and ccBlkUnindent
commands. teLPT is the line printer to use for the ccBlkPrint command
(defaults to 1, meaning LPT1). teDefExt is the default file extension.
teBlkBegin and teBlkEnd mark the start and end of the current block; the
block is undefined if teBlkBegin.Line = 0 or teBlkEnd.Line = 0.

teInsert         = $00000001;

!WRAP

teInsert..teNoCtrlZ and teInProcess are all identical to the corresponding
meXxx options for a Memo/MemoFile.

teNoRepeatGlobal indicates that the global search option ('G') should be
ignored when the repeat last find command (^L) is issued.

teSmartTabs indicates that smart tabs are to be used rather than fixed
tabs.

teSearching is a status flag that indicates that a search operation is
currently in progress. teSearchStart is a flag set at the start of a search
operation.

teSearchEnd is a flag set at the end of a search operation. These flags
make it possible for a user-written status routine to display a message
indicating that a search is in progress, or to display a message at the end
of a search-and-replace operation indicating the number of replacements
that were made (the count is stored in the teReplacements field of the
TextEditor).

teHighlightOn and teHighlightBack are internal flags used when highlighting
a string found in a search operation.

teMarkersOn indicates that all text markers in use should be displayed.
teBlockOn indicates that the currently marked block should be visible.

See also:  1050BadEditorOptions  1053DefEditorOptions  1029TextEditor.teOptionsOn

!TOPIC 1067 Commands,OpEditor
!NOINDEX
!NOSEARCH
OpEditor offers an extensive set of text editing commands, arranged by
category in the list below. Several of the commands in the list are enabled
only if one or more user-written procedures are in place. In each of these
cases, the entry for the command ends with the name(s) of the TextEditor
method(s) that provide the relevant programming hook(s). The names are also
cross-reference links to topics.

Note too that all of the file-related commands are disabled if the
TextEditor is being used as an advanced memo editor.

!LINE

is in effect.

!LINE
Cursor right one character.

!LINE

!LINE

!LINE

!LINE
Cursor to end of line.

ccUp          <Up>, <CtrlE>
!LINE
Cursor up one line.

ccDown        <Down>, <CtrlX>
!LINE
Cursor down one line.

ccScrollUp    <CtrlW>
!LINE

!LINE

Search and Replace

ccSearch      <CtrlQ><F>
!LINE
Allows the user to search for any string of up to 30 characters. After
entering a search string, the user is prompted for search options, which
include 'B' (search Backwards), 'G' (search Globally), 'L' (search Locally,
within marked block), and 'U' (ignore case). 1032SetEditProc

ccReplace     <CtrlQ><A>
!LINE
Works like ccSearch, except that the user is asked for a replacement string
of up to 30 characters. In addition to the search options available for
ccSearch, 'N' may be used to indicate that replacements are to be made
without asking for confirmation. If 'N' is not selected, each replacement
must be confirmed: yes (make the replacement), no (don't make it), all
(make this replacement and continue making replacements without further
confirmation), and quit (stop the whole operation). 1032SetEditProc,
1034SetYesNoProc

ccReSearch    <CtrlL>
!LINE
Repeat the last search or replace operation.

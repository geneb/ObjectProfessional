!TOPIC 1 Help on Help
!INDEX 1
!NOSEARCH
This pop-up reference POPHELP is a tool to help you make most effective use
of the many capabilities in Object Professional 1.0. While it cannot be a
complete replacement for the printed manual, it does serve as a valuable
quick reference to the available routines and the most important data
structures and constants.

POPHELP is a memory resident program that can be activated in several ways:

<ShiftF1>
!LINE
Reads a word from the screen at a location nearest to the cursor position
and then scans the list of topics and displays the matching topic. If none
is found, it displays the main help index. The help window is positioned
away from the cursor position if possible.

<ShiftF2>
!LINE
Redisplays the most recently selected help topic.

<ShiftF3>
!LINE
Pops up the main help index, which displays the names of the Object
Professional units and other items of interest, such as Help on Help.

The following keystrokes may be used after POPHELP is popped up:

<F1>
!LINE
Displays the main topic index again.

<AltF1>
!LINE
Displays the topic that was displayed prior to the current one.

<F3>
!LINE
Prompts for a new help file. POPHELP will read any help file built
using the Object Professional 1.0 MAKEHELP utility.

<F4>, <CtrlK><C>
!LINE
If a block is marked and visible, pastes it from the help system to the
underlying editor.

<F5>, <AltZ>
!LINE
Toggles the zoom mode of the window.

<F6>, <AltM>
!LINE
Enters move/resize mode--you can use the cursor keys to move the window
around the screen, or the Shift-cursor keys to resize the window. <PgUp>
and <PgDn> change the height of the window in increments of 4.
<CtrlLeftArrow> and <CtrlRightArrow> change the width in increments of 10.
Note that the window cannot be resized while it is zoomed.

<F7>, <CtrlK><B>
!LINE
Marks the beginning of a block.

<F8>, <CtrlK><K>
!LINE
Marks the end of a block.

<F9>, <CtrlL>
!LINE
Continues the last topic name search, using the search string obtained
with the <ShiftF1> hotkey. Searching is circular.

<CtrlK><H>
!LINE
Hides or unhides the currently marked block. <F4> will paste only if a
block is visible.

<PgUp>, <PgDn>
!LINE
Moves to the next or previous page of the current help topic.

<Left>, <Right>, <Up>, <Down>
!LINE
Moves the cursor around the help window. Positioning the cursor over a
cross-reference link and pressing <Enter> will load and display the
associated help topic.

<Tab>
!LINE
Jumps to the next cross reference topic, if any.

<ShiftTab>
!LINE
Jumps to the previous cross reference topic, if any.

<Enter>
!LINE
Selects the currently highlighted cross-reference topic.

<Esc>, <AltX>
!LINE
Exits POPHELP. The current topic and page are retained and may be
redisplayed by pressing <ShiftF2>.

!TOPIC 2 Defines
!INDEX 2
!NOSEARCH
A number of conditional defines affect the behavior of Object
Professional. All such defines are located in the file OPDEFINE.INC, which
is included by all units. By convention, unwanted defines are deactivated
by placing a period immediately after the opening comment brace, e.g.:

   {.$DEFINE UseMouse}

The following defines are contained in OPDEFINE.INC. The default condition
of the define is indicated by whether a period appears in the define.

{$N-}
  Controls whether the Float type in OpString is defined as an Extended
  or a Real. Also controls the inclusion of certain methods that apply only
  to 8087 numeric types.

{$DEFINE InitAllUnits}
  Forces all units to contain an initialization block, even those that do
  not have any actual initialization code. Adding these blocks helps to
  work around a bug in Turbo Debugger 1.5.

{.$DEFINE UseCrt}
  Determines whether or not OpCrt is compiled in such a way as to coexist
  peacefully with the standard Turbo Pascal CRT unit.

{$DEFINE UseMouse}
  Enables mouse support in various units.

{$DEFINE UseScrollBars}
  Enables scroll bar support in OpWindow and units that depend on it.

{$DEFINE UseHotSpots}
  Enables hot spot support in OpWindow and units that depend on it.

{$DEFINE UseShadows}
  Enables shadowing support in OpWindow.

{$DEFINE UseAdjustableWindows}
  Enables the AdjustWindow, ResizeWindow, MoveWindow, and other methods in
  OpWindow.

{$DEFINE FourByteDates}
  Dates in OpDate are stored as LongInt's when enabled, Word's otherwise.

{$DEFINE UseDates}
  Enables support for editing of dates in OpEdit and OpEntry. Also
  enables certain formatting options in OpDir.

{.$DEFINE UseBCD}
  Enables support for BCD reals in OpEdit, OpEntry, and OpForm.

{$DEFINE PickListFields}
  Enables support for pick list fields in OpEntry.

{$DEFINE FastDispose}
  Enables a special dispose technique that improves performance in
  OpSort. Disable if the program must allocate heap space during the
  sort. Note that FastDispose is automatically undefined under Turbo Pascal
  6, with which it is incompatible.

{$DEFINE ThwartSideKick}
  When defined, OpTsr tries to thwart SideKick.

{$DEFINE UseStreams}
  When defined, streams support is available for all objects.

{$DEFINE SupportXms}
  When defined, XMS support is enabled in OpSwap.

{.$DEFINE UseDrag}
  When defined, OpDrag support will be included in all units containing
  CommandWindow-based objects.

{.$DEFINE UseCalcEdit}
  If UseCalcEdit is defined, the new calculator-style editor will be
  compiled into OpFEdit under the name 'NumberEditor', and the standard
  numeric editor will be omitted from the unit.

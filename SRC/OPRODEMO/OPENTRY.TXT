!TOPIC 1136 OpEntry
OpEntry builds upon the groundwork laid by 1291OpField and
2617OpSelect to create a flexible data entry system. The following
hierarchy shows the objects implemented in OpEntry and, within brackets,
those that they are derived from:

[3045CommandWindow]
  [2618AbstractSelector]
    1137EntryScreen
      1214ScrollingEntryScreen

[0017AbstractField]
  [1292SelectField]
    [1303EntryField]
      1240PickField
      1241WindowField

See also:  1232Commands

!TOPIC 1137 EntryScreen
!NOINDEX
1136OpEntry.EntryScreen = object(2618AbstractSelector)
!LINE
EntryScreenPtr = ^EntryScreen;

The EntryScreen extends the concepts introduced in the AbstractSelector. It
not only handles cursor movement from field to field but also knows how to
perform data entry on those fields. It can automatically invoke the
appropriate routines to convert the raw data associated with fields into
displayable strings, to edit those strings, to validate their contents, and to
convert them back again.

EntryScreen provides methods in the following categories:

  1233Adding fields             1231Declarations            1236Delimiters
  1239General                   1238Modifying fields        1234Setting options
  1235Streams                   1237User hooks
;

!TOPIC 1222 Options,EntryScreens
!NOINDEX
!NOSEARCH
The following options are available for EntryScreens:

!NOWRAP
esScrollByPage     = $01;
esMousePage        = $02;
esReadOnly         = $04;
esSamePosition     = $08;
esUpdateActive     = $10;
esDeallocUserRec   = $40;
esFastUpdates      = $80;
!WRAP

esScrollbyPage affects the vertical scrolling behavior of a
ScrollingEntryScreen when the cursor is moved to a field not currently on
screen. If the option is off, the display will be scrolled only enough to
reveal the field to move to. If the option is on, the display will be scrolled
by a full page (usually equal to the number of rows that can be displayed at
one time).

esMousePage determines what happens when you click on the increment or
decrement hot spots of a scroll bar. If the option is off, clicking on these
spots is equivalent to pressing <Down> or <Up>, respectively. If the option is
on, clicking on them is equivalent to pressing <PgDn> or <PgUp>.

esReadOnly specifies that the EntryScreen should operate in "read-only" mode,
meaning that the user may not alter the contents of any of the fields.

esSamePosition is to be used in situations where you need to return to a field
that you just left (or started to leave) with the cursor at the same position
it was at previously.

esUpdateActive, esDeallocUserRec, and esFastUpdates are internal flags.

See also:  1224BadEntryOptions  1223DefEntryOptions  1234Setting options

!TOPIC 1232 Commands,OpEntry
!NOINDEX
!NOSEARCH
The editing and field movement commands supported by the EntryScreen are
arranged by category in the list below. In each case the first line gives
the name of the command, followed by the key(s) to which it is normally
assigned. The second and following lines give a brief description of the
command.

Because most of the commands behave differently depending on the field
editor in use, the descriptions are generally preceded by one or more
abbreviations, indicating the field editor being described. These are the
abbreviations used, followed by the complete name of the field editor, and
the corresponding methods:

!NOWRAP
SS  SimpleStringEditor  AddSimpleXxxField
SC  SimpleCharEditor    AddSimpleCharField, AddSimpleBooleanField,
                        AddSimpleYesNoField
SE  StringEditor        AddXxxField
CE  CharEditor          AddCharField, AddBooleanField, AddYesNoField
NE  NumberEditor        AddNumericXxxField
MC  ChoiceField.Edit    AddChoiceField
ML  MultiLine.Edit      AddMultiLineField
PE  PickField.Edit      AddPickStringField
!WRAP

Now, here are the commands recognized by the various field editors and by the
EntryScreen.

Cursor Movement

!NOWRAP
ccLeft        <Left>, <CtrlS>
!WRAP
SS, SE, ML: Cursor left one character. If the cursor is already at the
beginning of the field, and the efAutoAdvCurBegin option is on, the cursor
will be moved left one field.
!LINE
!NOWRAP
PE: Cursor left one item.
Else: Cursor left one field.

ccRight       <Right>, <CtrlD>
!WRAP
SS, SE, ML: Cursor right one character. If the cursor is already at the end of
the field, and the efAutoAdvCurEnd option is on, the cursor will be moved
right one field.
!LINE
!NOWRAP
PE: Cursor right one item.
Else: Cursor right one field.

ccWordLeft    <CtrlLeft>, <CtrlA>
SS, SE, ML: Cursor left one word.
Else: Cursor left one field.

ccWordRight   <CtrlRight>, <CtrlF>
SS, SE, ML: Cursor right one word.
Else: Cursor right one field.

ccHome        <Home>, <CtrlQ><S>
SS, SE: Cursor to beginning of field.
ML: Cursor to beginning of line.
PE: Cursor to first item in pick list.

ccEnd         <End>, <CtrlQ><D>
SS, SE: Cursor to end of field.
ML: Cursor to end of line.
PE: Cursor to last item in pick list.

ccUp          <Up>, <CtrlE>
ML: If cursor is not on top line of edit field, moves cursor up one line.
PE: Cursor up one item.

ccDown        <Down>, <CtrlX>
ML: If cursor not on bottom line of edit field, moves cursor down one line.
PE: Cursor down one item.

ccTab         <Tab>, <CtrlI>
Cursor to next subfield.

ccBackTab     <ShiftTab>
Cursor to previous subfield.

ccPageUp      <PgUp>
PE: Cursor up one page (within pick list).
Else: Cursor up one page (within entry screen).

ccPageDn      <PgDn>
PE: Cursor down one page (within pick list).
Else: Cursor down one page (within entry screen).

ccFirstFld    <CtrlPgUp>, <CtrlQ><R>
Cursor to first field.

ccLastFld     <CtrlPgDn>, <CtrlQ><C>
Cursor to last field.

Insertion and Deletion

ccDel         <Del>, <CtrlG>
SS, SE, NE, ML: Delete character at cursor.

ccBack        <Bksp>, <CtrlH>, <CtrlBksp>
SS, SE, ML: Delete character to left of cursor.
NE: Delete character at cursor.

ccDelLine     <CtrlY>
SS, SE, NE: Clear entire field.
ML: Delete current line.

ccDelEol      <CtrlEnd>, <CtrlQ><Y>
SS, SE: Delete from cursor to end of subfield.
ML: Delete from cursor to end of line.

ccDelBol      <CtrlHome>
SS, SE: Delete from beginning of subfield to cursor.
ML: Delete from beginning of line to cursor.

ccDelWord     <CtrlT>
SS, SE: Delete word to right of cursor.
!WRAP
ML: Delete word to right of cursor. If cursor is beyond the end of line, and
the following line will fit at the end of the current line, the following line
is joined with the current line.

!NOWRAP
ccCtrlChar    <CtrlP>
!WRAP
SS, SE, ML: Insert control character. For example, to insert a ^G, you would
enter <CtrlP><CtrlG>.

!NOWRAP
ccIns         <Ins>, <CtrlV>
!WRAP
SS, SE, ML: Toggle insert mode on and off. A fat cursor indicates insert mode;
a thin cursor indicates overtype mode.

!NOWRAP
Other

ccSelect      <Enter>, <CtrlM>, <ClickCenter>
SS, SC, SE, CE, NE, MC: Accept contents of field.
!WRAP
ML: If in overtype mode, moves cursor to beginning of next line. If in insert
mode, inserts new line at position of cursor and moves cursor to beginning of
new line (if possible).
!LINE
PE: Accept current item and advance cursor to the next field. Note that this

!NOWRAP
ccDone        <^Enter>, <CtrlJ>, <CtrlK><D>, <CtrlK><Q>
Accept contents of field.

ccMouseSel    <ClickLeft>
!WRAP
SS, SC, SE, CE, NE, MC: Move cursor to field pointed to by mouse cursor. If

!NOWRAP
ccRestore     <CtrlR>, <CtrlQ><L>

ccIncChoice   <+>, <Space>

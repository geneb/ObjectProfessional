!TOPIC 942 LineEditor
!NOINDEX
!NOWRAP
0941OpEdit.LineEditor = object(2306Root)
  ...
  leInsertMode  : Boolean;    {True if insert mode; False for overtype}
  leColors      : ColorSet;   {colors}
  ...
end;
LineEditorPtr = ^LineEditor;
!WRAP

Like the EntryScreen, the LineEditor object supplied in OpEdit works with
the various field objects derived from EntryField. The similarities end
there, however. A LineEditor is not descended from a CommandWindow, nor
does it work with multiple data entry fields. Its purpose is simply to
provide a high-level interface to the low-level field objects in OpField,
allowing them to be used to edit individual variables, as opposed to groups
of variables.

Two of LineEditor's fields are shown above: leInsertMode indicates whether
or not the LineEditor is currently in insert mode. leColors is a copy of
the ColorSet passed to the Init constructor.

!TOPIC 1013 Primary options,LineEditor
!NOINDEX
!NOSEARCH
Most of the primary edit options recognized by the LineEditor correspond
directly to the primary field options documented previously in the
OpField section. 'lePasswordMode', for example, means the same thing as
'efPasswordMode'. There are, however, several field options that are
inappropriate for a LineEditor, and there are also a couple (of secondary
options) that have meaning only for a LineEditor. These are the issues
addressed below.

!NOWRAP
leProtected        = $00000001;
leHidden           = $00000002;
leMapCtrls         = $00000004;
leMultChoice       = $00000008;
leInvisible        = $00000010;
!WRAP

With the exception of leMapCtrls, these field options are inappropriate for
line editing. leMapCtrls is equivalent to the efMapCtrls option in
OpField.

!NOWRAP
leAutoAdvCharBegin = $00000020;
leAutoAdvCharEnd   = $00000040;
leAutoAdvCurBegin  = $00000080;
leAutoAdvCurEnd    = $00000100;
leAutoAdvanceChar  = leAutoAdvCharBegin+leAutoAdvCharEnd;
leAutoAdvanceCursor= leAutoAdvCurBegin+leAutoAdvCurEnd;
leAutoAdvanceBegin = leAutoAdvCharBegin+leAutoAdvCurBegin;
leAutoAdvanceEnd   = leAutoAdvCharEnd+leAutoAdvCurEnd;
leAutoAdvance      = leAutoAdvanceBegin+leAutoAdvanceEnd;
leInsertPushes     = $00000200;
leRightJustify     = $00000400;
lePasswordMode     = $00000800;
leCursorToEnd      = $00001000;
leTrimBlanks       = $00002000;
leClearFirstChar   = $00004000;
leForceOvertype    = $00008000;
leForceMode        = $00010000;
leClickExit        = $00020000;
leBeepOnError      = $00040000;
leAutoNumLock      = $00080000;
leParensForMinus   = $00100000;
!WRAP

These options all have the same meaning as their counterparts in OpField
(which have a prefix of 'ef' rather than 'le').

!NOWRAP
leReadOnly         = $00800000;
leHouseCursorAtEnd = $01000000;
leForceUpper       = $02000000;
leForceLower       = $04000000;
!WRAP

If leReadOnly is set, the user may move the cursor around in the edit field,
but may not modify its contents. The other options apply only to the ReadXxx
and YesOrNo methods. See the earlier section on read methods that use
SimpleStringEditor.

!NOWRAP
leAllowEscape      = $08000000;
leShowReadChar     = $10000000;
leHideCursor       = $20000000;
leDefaultAccepted  = $40000000;
leMapExtended      = $80000000;
!WRAP

These options apply only to the ReadChar and YesOrNo methods.

See also:  0947LineEditor.leEditOptionsOn  1014Secondary options

;------------------------------------------------------------
!TOPIC 1014 Secondary options,LineEditor
!NOINDEX
!NOSEARCH
The secondary edit options shown below correspond to the secondary field
options documented in the OpField section ('sef' prefix). Unlike most of
the primary options, however, most of the secondary options have particular
significance for a LineEditor.

!NOWRAP
sleSmartExponents  = $02000000;
sleSwitchCommands  = $04000000;
slePadCurrentOnly  = $08000000;
sleSuppressZero    = $10000000;
!WRAP

sleSmartExponents selects alternate behavior for fields using the 'E'
picture mask character: rather than always using scientific notation,
decimal notation will be used if possible. See the entry in the manual for
the SetPadChar method for a discussion of slePadCurrentOnly. The
sleSwitchCommands and sleSuppressZero options, which have the same meanings
as their counterparts in OpField, are inappropriate for line editing.

!NOWRAP
sleNoFieldMovement = $20000000;
sleWindowRelative  = $40000000;
sleInsertByDefault = $80000000;
!WRAP

See also:  0950LineEditor.leSecEditOptionsOn  1013Primary options

!TOPIC 1021 Commands,LineEditor
!NOINDEX
!NOSEARCH
The editing commands supported by the LineEditor are arranged by category
in the list below. In each case the first line gives the name of the
command, followed by the key(s) to which it is normally assigned. The
second and following lines give a brief description of the command. Because
most of the commands behave differently depending on the field editor in
use, the descriptions are generally preceded by one or more abbreviations,
indicating the field editor being described. These are the abbreviations
used, followed by the complete name of the field editor, and the
corresponding LineEditor methods:

!NOWRAP
SS  SimpleStringEditor  ReadXxx
SC  SimpleCharEditor    ReadChar, YesOrNo
SE  StringEditor        EditXxx
CE  CharEditor          EditChar, EditBoolean, EditYesNo
NE  NumberEditor        EditNumericXxx
MC  ChoiceField.Edit    EditChoice
ML  MultiLine.Edit      EditMultiLine
!WRAP

The behavior of most of the cursor movement commands further depends on the
state of the sleNoFieldMovement, which is on by default. If the option is
in effect, no field movement command (ccLeft, ccRight, ccWordLeft,
ccWordRight, ccUp, ccDown) can produce an exit. If the option is turned
off, these commands will be treated just as they would if the field was
part of an EntryScreen.

Cursor Movement

ccLeft        <Left>, <CtrlS>
!LINE
SS, SE, ML: Cursor left one character.

ccRight       <Right>, <CtrlD>
!LINE
SS, SE, ML: Cursor right one character.

ccWordLeft    <CtrlLeft>, <CtrlA>
!LINE
SS, SE, ML: Cursor left one word.

ccWordRight   <CtrlRight>, <CtrlF>
!LINE
SS, SE, ML: Cursor right one word.

ccHome        <Home>, <CtrlQ><S>
!LINE
SS, SE: Cursor to beginning of field.
!LINE
ML: Cursor to beginning of line.

ccEnd         <End>, <CtrlQ><D>
!LINE
SS, SE: Cursor to end of field.
!LINE
ML: Cursor to end of line.

ccUp          <Up>, <CtrlE>
!LINE
ML: If cursor is not on top line of edit field, moves cursor up one line.

ccDown        <Down>, <CtrlX>
!LINE
ML: If cursor not on bottom line of edit field, moves cursor down one line.

Insertion and Deletion

ccDel         <Del>, <CtrlG>
!LINE
SS, SE, NE, ML: Delete character at cursor.

ccBack        <Bksp>, <CtrlH>, <CtrlBksp>
!LINE
SS, SE, ML: Delete character to left of cursor.
!LINE
NE: Delete character at cursor.

ccDelLine     <CtrlY>
!LINE
SS, SE, NE: Clear entire field.
!LINE
ML: Delete current line.

ccDelEol      <CtrlEnd>, <CtrlQ><Y>
SS, SE: Delete from cursor to end of subfield.
!LINE
ML: Delete from cursor to end of line.

ccDelBol      <CtrlHome>
!LINE
SS, SE: Delete from beginning of subfield to cursor.
!LINE
ML: Delete from beginning of line to cursor.

ccDelWord     <CtrlT>
!LINE
SS, SE: Delete word to right of cursor.
!LINE
ML: Delete word to right of cursor. If cursor is beyond end of line, and
following line will fit at end of current line, the following line is joined
with the current line.

ccCtrlChar    <CtrlP>
!LINE
SS, SE, ML: Insert control character. For example, to insert a ^G, you
would enter <CtrlP><CtrlG>. This command is not available if the video
attribute for control characters is the same as that for ordinary text and
the leMapCtrls option is on.

ccIns         <Ins>, <CtrlV>
!LINE
SS, SE, ML: Toggle insert mode on and off. A fat cursor indicates insert
mode; a thin cursor indicates overtype mode.

Other

ccSelect      <Enter>, <CtrlM>, <ClickCenter>
!LINE
SS, SC, SE, CE, NE, MC: Accept contents of field.
!LINE
ML: If in overtype mode, moves cursor to beginning of next line. If in
insert mode, inserts new line at position of cursor and moves cursor to
beginning of new line (if possible).

ccDone        <^Enter>, <CtrlJ>
!LINE
Accept contents of field.

ccMouseSel    <ClickLeft>
!LINE
SS, SC, SE, CE, NE, MC: Accept contents of field.
!LINE
ML: If mouse within field, moves cursor to position of mouse.

ccRestore     <CtrlR>, <CtrlQ><L>
!LINE
SS, SC, SE, CE, NE, MC: Restore original contents of the field.
!LINE
ML: Restore original contents of current line.

ccIncChoice   <+>, <Space>
!LINE
MC: Increment a multiple choice field. This command is meaningful only if
EditChoice was called. In all other cases, <+> and <Space> will be treated
as ordinary characters.

ccDecChoice   <->
!LINE
MC: Decrement a multiple choice field. This command is meaningful only if
EditChoice was called. In all other cases, <-> will be treated as an
ordinary character.

ccHelp        <F1>, <ClickBoth>
!LINE
Help. If a user-written help routine has been established by calling
EditCommands.SetHelpProc, pressing <F1> will call that routine; otherwise
this command does nothing. If there is a GetHelpProc, the line editor will
pass it the following three parameters: ucEdit, nil, and the value
designated as the line editor's help index (see LineEditor.SetHelpIndex),
which defaults to 0.

ccQuit        <Esc>, <CtrlBreak>, <ClickRight>
!LINE
Quit editing.

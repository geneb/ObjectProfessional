!TOPIC 2127 OpPrint
Whereas 2211OpPrnLow focused on transferring characters to the printer
and reading back the status, OpPrint deals with the appearance of printed
characters on the page. OpPrint's primary object, Printer, provides the
power needed to change fonts, enable alternate printing modes, modify lines
per inch, draw lines and boxes--basically to take advantage of all the
features of any printer.

  2128Printer                   2206PrintMode
;
;------------------------------------------------------------
!TOPIC 2128 Printer
!NOINDEX
2127OpPrint.Printer = object(2306Root)
!LINE
PrinterPtr = ^Printer;

The Printer object is a large one, with over 50 documented methods and
another 70 or so for internal use. If the printer you wish to access is
already supported by 0554OpDevice, the apparent complexity of Printer
drops immediately, since many of the methods are intended for
initialization of specific printer characteristics, and OpDevice takes
care of that for you. The remaining methods are used primarily for enabling
and disabling print modes, positioning the printhead, and writing strings.

!TOPIC 2188 Print mode options
!NOINDEX
!NOSEARCH
!NOWRAP
pmFont              = $0001;
pmChangesLPI        = $0004;
pmChangesLPP        = $0008;
pmChangesCPI        = $0010;
pmChangesPoint      = $0020;
pmChangesTopMargin  = $0040;
pmChangesLandscape  = $0080;
pmChangesStrike     = $0100;
!WRAP

These are the options that apply to each PrintMode added to a Printer. One
or more of these values are passed to the AddPrintMode method.

pmFont indicates that the mode refers to a non-toggling font (e.g., bold
roman on a LaserJet).

pmChangesLPI, pmChangesLPP, pmChangesCPI, pmChangesPoint, and
pmChangesTopMargin indicate that the mode changes the indicated
characteristic of the page or character dimensions. The appropriate
dimensional value from the PrintMode object will update the current value
of LPI, LPP, CPI, Point, or TopMargin in the Printer object when such a
mode is turned on.

pmChangesLandscape indicates that the mode changes the printer orientation,
which again updates a flag internal to the Printer object when enabled.
pmChangesStrike indicates that the mode changes the double-strike behavior
of the printer.

See also:  2162Printer.AddPrintMode

!TOPIC 2192 Printer options
!NOINDEX
!NOSEARCH
!NOWRAP
pPositionable       = $0001;
pBackSpaceAllowed   = $0002;
pNoFormFeed         = $0004;
pSoftFormFeed       = $0008;
pSupportsHPRules    = $0010;
pUseDeciPoints      = $0020;
pNoFontStack        = $0040;
pNoSortPositionable = $0080;
pLJColAdjust        = $0100;
!WRAP

These option flags apply to a Printer as a whole.

pPositionable indicates that the physical printer's internal cursor can be
positioned randomly on the page; this generally applies to laser printers
only.

pBackSpaceAllowed indicates that the printer interprets ^H as a command to
move the printhead one column to the left.

If pNoFormFeed is enabled, Printer ignores calls to the FormFeed method;
this may be useful when generating draft versions of reports.

If pSoftFormFeed is enabled, Printer ejects a page by printing enough
CR/LF's to fill out the page; otherwise it just writes a form feed
character.

pSupportsHPRules indicates that the printer supports the Hewlett-Packard
LaserJet line and box drawing commands. This option enables extra
functionality in the OpForm unit.

If pUseDeciPoints is set and a LaserJet printer is registered (see
OpDevice) coordinates passed to Position and PrintStrPos methods are
interpreted in decipoints rather than in rows and columns.

pNoFontStack specifies that a font stack should not be used.

If you are using OpForm to print a form on a printer is positionable, and
the form has no directives (added with PrintedForm.AddDirective), printing
can be sped up significantly by setting the pNoSortPositionable option for
the Printer object associated with the form.

The pLJColAdjust option is used to correct an off-by-one problem that occurs
with LaserJet printers (see the READ.1ST file for details).

See also:
    2173Printer.pOptionsAreOn      2172Printer.pOptionsOff
    2171Printer.pOptionsOn

!TOPIC 2193 PrintMode codes
!NOINDEX
!NOSEARCH
!NOWRAP
DoubleStrike        = 255;
PrinterReset        = 254;
Italics             = 253;
Bold                = 252;
Dim                 = 251;
